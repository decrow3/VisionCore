# Temporal Decoding Diagnostic Plan for FEM / Hyperacuity Analysis

## Purpose

This document lays out a concrete diagnostic and implementation plan for evaluating whether the current `C ≈ A` result in the temporal decoding analysis reflects a genuine scientific null or a methodological artifact introduced by the representation pipeline.

The core issue is that the current analysis may be destroying a real temporal or spatially structured signal before the decoder ever sees it. The plan below is designed to move the project from **interpreting a result** to **verifying a measurement**.

---

## Executive Summary

There are four high-priority risks in the current analysis pipeline:

1. **Spatial collapse may be discarding the signal**
   - The canonical model utilities return a **spatial rate map** per neuron, not a scalar.
   - If the temporal code from FEMs is expressed as movement of a hotspot within the readout map, collapsing with `amax` may remove the informative structure.

2. **Temporal integration may be artificially shortened**
   - If all trials are truncated to the minimum trace length, a few short traces may determine the integration window for the entire decoding analysis.

3. **Stimulus normalization may be inconsistent across pipelines**
   - The trained digital twin expects a specific normalization convention.
   - A mismatch between the old stack-based pipeline and the new hi-res differentiable pipeline could flatten responses or alter comparisons.

4. **The recurrent / temporal evaluation path may not match the intended interpretation**
   - The analysis currently uses lag-embedded inputs and reads out the last temporal slice.
   - This is not automatically equivalent to a continuously stateful recurrent rollout over a long eye trace.

5. **Integration-time decoder representation can destroy signal**
  - A common integration-time implementation is to flatten the last W frames ($W \times N$) and run PCA before decoding.
  - In the hyperacuity regime this can artificially obliterate signal (e.g., even stabilized can look near-chance and flat with W), producing a misleading “no accumulation” conclusion.
  - The accumulation-aligned control is to decode from the mean rate over the last W frames (N-dim) and verify that accuracy increases with W under real FEM.

The first goal is not to redesign the science story. The first goal is to determine whether the representation itself is trustworthy.

---

## Guiding Principle

The key question is:

> Does the current analysis preserve the spatially structured and temporally evolving model response generated by FEMs, or does it collapse that structure into a representation that is incapable of showing a temporal advantage?

The edits and tests below are organized to answer that question directly.

## Priority Order

The recommended order is:

1. Stop hiding the spatial map
2. Visualize the map dynamics directly
3. Measure shift sensitivity in a minimal controlled test
4. Fix temporal-window handling
5. Enforce normalization consistency
6. Verify equivalence between stimulus pipelines
7. Re-run a small decoding subset with alternative representations
8. Only then revisit scientific interpretation

# Part I. Code Edits to Make

## Edit 1. Add an explicit spatial-collapse dispatcher

### Motivation

The current model path naturally produces a spatial rate map of shape `(B, N, H, W)`. The downstream decoding analysis needs a representation, but the choice of representation is currently likely underconstrained.

If `amax` is used, it may introduce position invariance and discard the very effect that eye movements create.

### Goal

Make the spatial-collapse operation a configurable analysis choice rather than a silent hard-coded step.

### Recommended implementation

Create a helper like:

```python
def collapse_rate_map(rate_map, mode="max", spatial_pca=None):
    '''
    rate_map: (B, N, H, W)

    modes:
      - "max":  return (B, N)
      - "mean": return (B, N)
      - "flat": return (B, N*H*W)
      - "pca":  flatten then project to low-dimensional representation
    '''
```

### Suggested file location

- `scripts/temporal_decoding/rate_computation.py`
- or a small new helper module:
  - `scripts/temporal_decoding/representations.py`

### Required modes

- `max`
- `mean`
- `flat`
- `pca`

### Why this matters

This is the single most important edit. If the apparent null depends strongly on the collapse mode, the original interpretation cannot be trusted.

### Motivation

Before arguing about PCA, decoding, or information theory, inspect the raw maps directly.

If FEMs cause the hotspot within a neuron's `(H, W)` response map to move over time while the peak amplitude stays similar, then `amax` is removing signal by construction.

### Goal

Visualize the time evolution of the spatial response map for:
- one neuron
- one trial
- one stimulus condition

### Recommended implementation

Create a helper like:

```python
def inspect_single_trial_maps(rates, neuron_idx, times):
    '''
    rates: (T, N, H, W)
    neuron_idx: int
    times: list of frame indices
    '''
```

This should generate a panel of `imshow` frames across time.

### Suggested file location

- `scripts/temporal_decoding/inspect_maps.py`
- or inside a notebook / diagnostic script at first

### Conditions to inspect

- real FEM
- stabilized
- matched null

### Why this matters

This is the "seeing-is-believing" diagnostic. It directly answers whether the model is expressing the effect spatially.

## Edit 3. Add a controlled step-shift sensitivity test

### Motivation

Real eye traces are complicated. Before interpreting them, test whether the model and readout are sensitive to a simple controlled shift.

### Goal

Create a minimal test where:
- one stimulus is static
- the same stimulus undergoes a single step shift at time `t0`
- the resulting response difference is measured at the level of:
  - raw spatial maps
  - collapsed scalar representation

### Recommended implementation

Create a diagnostic script like:

```python
def step_shift_test(model, readout, stim_stack, shift_px=1, t0=60, collapse="max"):
    '''
    Compare model responses to static vs single-step shifted stimuli.
    '''
```

### Suggested file location

- `scripts/temporal_decoding/step_shift_test.py`

### Interpretive logic

If the 1-pixel shift:
- is visible in raw maps but disappears after `amax`, the collapse is the bottleneck
- is invisible even in raw maps, the model is not sensitive at that scale
- produces a transient only at larger shifts, then the real FEM regime may simply be too fine or too smoothed

### Why this matters

This is the cleanest possible test of where sensitivity is being lost.

---

## Edit 4. Replace minimum-length truncation with explicit target windows

### Motivation

Using the shortest trial in the dataset to define the time window is too brittle. A few bad traces can force the whole analysis into a short-glimpse regime.

### Goal

Set a fixed target integration length and either:
- drop short trials
- or analyze multiple fixed windows deliberately

### Recommended implementation

Replace behavior like:

```python
T_use = min([r.shape[0] for r in rates_real])
```

with a function that:
- prints the length distribution
- uses an explicit `target_length`
- drops trials shorter than `target_length`
- reports how many trials were removed

### Suggested file location

- wherever `prepare_rate_tensors()` currently lives
- likely `scripts/temporal_decoding/decoding.py` or `rate_computation.py`

### Recommended initial values

- 120 frames
- 240 frames

depending on the frame rate and computational cost

### Why this matters

The temporal window is itself a scientific parameter. It should not be silently determined by the shortest trace.

---

## Edit 5. Centralize stimulus normalization

### Motivation

The digital twin was trained with a specific normalization convention. The old and hi-res pipelines must match that convention if their results are to be compared.

### Goal

Ensure all inputs passed to the twin are normalized in a consistent and explicit way.

### Recommended implementation

Create a helper like:

```python
def normalize_for_twin(stim, from_uint8=False):
    '''
    Return model-compatible input in approximately [-0.5, 0.5].
    '''
```

### Suggested file location

- `scripts/temporal_decoding/stimulus_utils.py`
- or next to other stimulus generation helpers

### Required checks

Support both:
- uint8-like `[0, 255]`
- float-like `[0, 1]`

### Why this matters

A mean-luminance shift can silently move the model into a flatter activation regime and suppress sensitivity to small stimulus changes.

---

## Edit 6. Add a pipeline-equivalence diagnostic

### Motivation

The stack-based and hi-res pipelines are only scientifically comparable if they produce functionally similar inputs and model responses in a shared regime.

### Goal

Compare both pipelines at an easy, comfortably resolved stimulus size.

### Recommended implementation

Create a script like:

```python
check_pipeline_equivalence.py
```

that compares the two pipelines at:
- LogMAR 0.4
- optionally LogMAR 0.6

### Compare

- stimulus intensity histogram
- mean and variance of input frames
- raw model response magnitude
- Model A decoding accuracy

### Criteria

If the easy-regime outputs differ materially, cross-pipeline comparisons in the hyperacuity regime are not safe.

### Why this matters

This isolates stimulus-generation and normalization mismatches before they contaminate interpretation.

---

## Edit 7. Make representation choice explicit in decoding

### Motivation

Once the above diagnostics are in place, decoding should explicitly compare representations rather than assuming one.

### Goal

Re-run a small decoding subset with:
- `max`
- `mean`
- `flat`
- `pca`

### Suggested approach

Use a small pilot:
- a few stimuli
- a few traces
- real FEM and stabilized
- one or two LogMAR values near the current threshold regime

### Why this matters

If `C ≈ A` depends strongly on representation choice, the current null is a representational artifact, not a scientific result.

# Part II. Tests to Run

## Test 1. Spatial hotspot inspection

### Purpose

Determine whether FEMs cause the hotspot within the spatial map to move over time.

### Procedure

For one neuron and one trial:
- plot several time points of the `(H, W)` map under real FEM
- repeat for stabilized
- optionally repeat for matched null

### Outcomes

- **Hotspot moves in real FEM but not stabilized**
  - strong evidence that spatial map dynamics carry the signal
- **Hotspot does not move**
  - less support for the spatial-collapse concern
- **Hotspot moves but `amax` is flat**
  - strong evidence that `amax` is discarding signal

### Interpretation

This is the most direct test of whether the representation is the problem.

---

## Test 2. Step-shift sensitivity

### Purpose

Measure whether the pipeline is sensitive to a controlled shift.

### Procedure

Compare static vs single-step-shifted input and examine:
- raw spatial maps
- collapsed representation

### Outcomes

- **Visible in maps, disappears after `amax`**
  - collapse is the bottleneck
- **Invisible in maps**
  - model / blur / stimulus regime may be too insensitive
- **Visible only at large shifts**
  - real FEMs may be too small or too smoothed to matter in the current setup

### Interpretation

This test separates model sensitivity from decoder representation.

---

## Test 3. Trial-length distribution and target-window robustness

### Purpose

Determine whether the temporal analysis is being constrained by a few short traces.

### Procedure

Print:
- min
- median
- 10th percentile
- 90th percentile
- number of traces shorter than each candidate window

Then re-run analysis with:
- 120-frame target
- 240-frame target

### Outcomes

- **Current minimum length is much shorter than the median**
  - current analysis likely loses temporal information
- **Results change substantially with target window**
  - current conclusions are window-dependent
- **Results are stable**
  - truncation is probably not the main issue

### Interpretation

This test determines whether the analysis is inadvertently in a short-glimpse regime.

---

## Test 4. Pipeline equivalence at easy LogMAR

### Purpose

Determine whether old and hi-res pipelines are functionally comparable.

### Procedure

At LogMAR 0.4:
- generate stimulus through both pipelines
- normalize both
- compare input statistics and Model A accuracy

### Outcomes

- **Outputs and accuracies agree closely**
  - cross-pipeline comparison is more defensible
- **Systematic mismatch**
  - fix normalization / blur / interpolation before interpreting hyperacuity results

### Interpretation

Do not trust cross-regime conclusions unless this test passes.

---

## Test 5. Representation comparison in decoding

### Purpose

Determine whether the null depends on spatial representation choice.

### Procedure

Run a small decoding subset with:
- `max`
- `mean`
- `flat`
- `pca`

### Outcomes

- **Only `max` gives `C ≈ A`**
  - the original null is likely artificial
- **All representations give `C ≈ A`**
  - stronger support for a true null
- **Spatially richer representations rescue C**
  - signal exists but was being discarded

### Interpretation

This is the decisive test for the representation hypothesis.

# Part III. Interpretive Decision Tree

## Case A. Maps move, `amax` hides it

**Conclusion:** The current null is a representation artifact.

What to say:
- The decoder representation was too invariant to capture FEM-induced spatial dynamics.
- The analysis must be re-run with a spatially richer representation.

---

## Case B. Maps do not respond even to small controlled shifts

**Conclusion:** The model / stimulus / blur regime may be insensitive at the tested scale.

What to say:
- The twin may not support the hypothesized hyperacuity effect under the current stimulus-generation assumptions.
- This is a model-sensitivity limitation, not yet a statement about biological coding.

---

## Case C. Easy-regime equivalence fails across pipelines

**Conclusion:** The stimulus-generation pipelines are not directly comparable.

What to say:
- Hyperacuity and standard-regime analyses cannot be interpreted together until normalization and resampling are unified.

---

## Case D. Representation is varied, diagnostics pass, and `C ≈ A` remains

**Conclusion:** The null becomes much more credible.

What to say:
- Under multiple reasonable representations and after validating sensitivity, temporal residual structure does not add discriminative information beyond the simpler rate-based representation.

Only in this case should `C ≈ A` begin to be interpreted as a scientific result.

---

# Part IV. Recommended First Patch

The first patch should focus on representation and sensitivity, not on final decoder sophistication.

## Implement first

1. `collapse_rate_map(...)`
2. `inspect_single_trial_maps(...)`
3. `step_shift_test.py`

## Then run first

1. hotspot visualization
2. step-shift sensitivity
3. small decoding comparison with `max` vs `mean`

## Why this is the right first patch

These changes require relatively little plumbing, but they provide the fastest path to determining whether the current null is being caused by:
- position-invariant collapse
- lack of model sensitivity
- or a genuinely absent temporal advantage

# Part V. Practical Notes for Interpretation

## What should be treated as provisional right now

The statement:

> "C ≈ A, therefore temporal structure adds nothing"

should be treated as **provisional** until the representation diagnostics above are completed.

A safer current statement is:

> Under the current downstream representation, especially the current spatial collapse, temporal residual features are not improving decoding.

That is a methodological observation, not yet a biological conclusion.

## What may already be more trustworthy

The comparison:
- **real FEM vs matched null**

may still be informative, especially if both are passed through the same pipeline. It is more robust than broad conclusions about the existence or absence of temporal code.

Still, it should be interpreted with caution until the main representation bottlenecks are checked.

# Bottom Line

The current priority is not to explain the `C ≈ A` result. The priority is to determine whether the analysis pipeline is capable of seeing the effect it claims to test.

If the spatial map carries informative FEM-induced motion and the downstream collapse erases it, then the null is artificial.  
If the model is genuinely insensitive at the relevant scale, then the question shifts from decoding to model validity.  
If the diagnostics pass and `C ≈ A` still holds, only then is the scientific null on solid ground.
